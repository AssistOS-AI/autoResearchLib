# DS002 - Data Model

## Introduction
This specification defines the formal objects that make the framework auditable. The point of the data model is not only serialization convenience. It is the mechanism that preserves the distinction between evidence, local theory, neighborhood structure, and retained commitment. If these objects are underspecified, the architecture collapses back into opaque text-to-label behavior.

## Core Content
The observational family for a report `x` is `O(x) = {O0, O1, ..., On}`. In the current implementation each observational hypothesis is a plain serializable object containing the source text, a normalized text view, tokenization, explicit cues, inferred cues, a `supportByDomain` map, an ambiguity list, a confidence profile, and an optional `focusDomain`. A cue is the smallest typed unit that can later affect theory induction. Every cue records its identifier, type, specificity, weight, visibility policy, supporting source spans, and whether it was explicit in the text or inferred by a rule. The current lift computes domain support operationally as `support(d) = 0.3 * genericSignal + explicitSignal(d) + 0.7 * inferredSignal(d)`. The base hypothesis `O0` keeps all detected cues, while focused hypotheses retain generic cues plus the cues of domains whose support remains within sixty percent of the best support under the current budget.

For a retained observational hypothesis `Oi` and a domain bundle `d`, the induced base theory is `T(Oi, d) = (Sigma, I, R, C, S)`. `Sigma` is the domain state schema, `I` is the current invariant set, `R` is the typed rewrite template family, `C` is the composition discipline, and `S` is a named score profile. The score profile is fully operational in the current code. Let `genericNorm`, `domainNorm`, `inferenceNorm`, `sequenceCoverage`, `supportShare`, and `focusBonus` be the normalized generic signal, explicit domain signal, inferred domain signal, matched state-sequence coverage, share of domain support, and focused-hypothesis bonus for the selected theory. The score dimensions are then computed as `evidenceCoverage = clamp(0.3 * genericNorm + 0.7 * domainNorm)`, `predictiveAdequacy = clamp(0.45 * sequenceCoverage + 0.35 * domainNorm + 0.2 * inferenceNorm)`, `compressionUtility = clamp(0.5 + 0.15 * genericNorm + 0.2 * supportShare + 0.15 * sequenceCoverage - 0.05 * (1 - sequenceCoverage))`, `compositionalSharpness = clamp(0.45 * sequenceCoverage + 0.3 * domainNorm + 0.15 * focusBonus + 0.1 * supportShare)`, `stability = clamp(0.45 + 0.2 * genericNorm + 0.15 * sequenceCoverage + 0.15 * supportShare - 0.1 * (1 - supportShare))`, and `alignmentUtility = clamp(0.45 * domainNorm + 0.2 * focusBonus + 0.2 * inferenceNorm + 0.15 * supportShare)`.

The current scalar total is not a replacement for the named dimensions; it is only the summary used for ranking and entropy calculations. The weighted total is `0.22 * evidenceCoverage + 0.28 * predictiveAdequacy + 0.10 * compressionUtility + 0.16 * compositionalSharpness + 0.14 * stability + 0.10 * alignmentUtility`. These weights keep predictive adequacy and evidence coverage primary while still preserving the practical value of compression, compositional clarity, stability, and downstream alignment.

| Score dimension | Operational meaning | Weight in total |
| --- | --- | --- |
| `evidenceCoverage` | Fraction of the available signal explained by the theory | `0.22` |
| `predictiveAdequacy` | Compatibility of the current theory with the matched state evolution | `0.28` |
| `compressionUtility` | Structured summary value without gratuitous local complexity | `0.10` |
| `compositionalSharpness` | Clarity of typed rewrites and admissible compositions | `0.16` |
| `stability` | Resistance to perturbation and later evidence updates | `0.14` |
| `alignmentUtility` | Utility for later conceptual alignment or explanation | `0.10` |

The local neighborhood for one observational hypothesis is `N(Oi) = (Ti, Mi, Ei, Qi)`. `Ti` is not only the set of base theories; it is the expanded family obtained by applying typed local moves to each base theory. The current implementation materializes three nearby variants for every base theory: `refined`, `coarsened`, and `refactorized`. Refinement sharpens predictive adequacy and compositional sharpness at the cost of compression. Coarsening improves compression and stability at the cost of predictive adequacy and sharpness. Refactorization keeps the theory near the base theory while slightly improving compositional and alignment terms. `Mi` is the explicit transformation set that records these moves. `Ei` is the family of observer-relative equivalence classes computed from visible cue identifiers and the visible state sequence. `Qi` stores robust consequences as the intersection of invariants across the retained frontier and theory-sensitive consequences as the invariant remainder of each theory.

The active commitment object is the frontier `F`. The strict frontier is the set of non-dominated theories under the named score dimensions. In the current implementation `A` dominates `B` exactly when every score dimension of `A` is at least the corresponding dimension of `B` and at least one dimension is strictly greater. The practical frontier is then widened by one rescue rule: if an entire domain disappears from the strict frontier but its best surviving theory is within a total-score tolerance of `0.08` from the best strict-frontier score, the best theory for that domain is reintroduced. The resulting frontier is sorted by total score and capped by the current frontier budget, which defaults to eight theories. This is the operational meaning of meta-rational retention in the repository: dominance prevents clutter, while rescue prevents premature elimination of still-plausible local families.

Questioning is also explicit. A `DiscriminatingQuestion` stores the prompt, the answer map by domain, the induced answer partition over the frontier, and the estimated information gain. The current question selector computes prior frontier entropy from normalized total scores, computes the expected entropy of each answer partition, and chooses the question with maximal information gain. A `FrontierUpdate` stores the selected question, the observed answer, score deltas, and the updated frontier summary. Matching answers increase predictive adequacy, stability, and alignment; conflicting answers reduce evidence coverage, predictive adequacy, compositional sharpness, stability, and alignment; unknown answers incur a smaller penalty. After the deltas are applied, the neighborhood is resummarized: frontier retention is recomputed, equivalence classes are rebuilt, and theory/domain entropies are updated.

Experiment reporting depends on the same objects. An experiment trace records the observational hypothesis family, the induced base theories, the retained frontier, the recommended question if any, the observed answer, and the updated frontier. The article should therefore be able to consume the experiment outputs without inventing a second interpretation layer. The data model is meant to be the common spine of the library API, experiment artifacts, and article narrative.

The stable external serialization of these objects should be a canonical CNL rather than a repository-private object schema. In the current repository the canonical usage layer serializes source context, observational statements, hypotheses, theories, transforms, equivalence classes, score lines, consequences, questions, updates, and frontier membership as deterministic keyword-led lines. That CNL is not a second model. It is a stable projection of the same auditable objects described above.

## Conclusion
The data model is the backbone of the framework because it keeps the architecture from dissolving into prose. The same objects should power the library API, the experiment outputs, and the generated article without manual reinterpretation.

### Critical Implementation Directives
1. Export serializable plain objects so experiment scripts can write JSON and article builders can consume the results directly.
2. Keep score dimensions explicit and named in every theory and neighborhood artifact, even when a weighted total is also computed.
3. Represent typed transformations, equivalence classes, frontier rescue, discriminating questions, frontier updates, and canonical CNL projections explicitly.
