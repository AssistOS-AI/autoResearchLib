# Project Decision Log

| Date | ID | Decision | Rationale | Impact | Status |
| --- | --- | --- | --- | --- | --- |
| 2026-04-14 | DEC001 | Keep the runtime dependency-free. | The prompt requires a standalone Node.js library. The first implementation can validate the architecture with deterministic observers and domain bundles, without outsourcing core behavior to third-party packages. | The code must rely on built-in Node.js APIs, including tests, file generation, and article build tooling. | Approved |
| 2026-04-14 | DEC002 | Restrict `AGENT.md` to repository policy. | The previous version mixed operational instructions with research specification material, which made the repository contract and the scientific design harder to maintain. | All technical substance is moved to DS files, while `AGENT.md` only describes maintenance rules. | Approved |
| 2026-04-14 | DEC003 | Treat meta-rational retention as an implementation policy, not as a new theorem. | The project can operationalize bounded non-dominated retention and observer-relative equivalence without claiming unsupported mathematical novelty. | Documentation and article prose stay grounded in auditable computation rather than inflated formal claims. | Approved |
| 2026-04-14 | DEC004 | Use plain Markdown notation in DS files and explicit HTML rendering in the article. | Standard Markdown does not guarantee LaTeX rendering, and the previous documentation relied on notation that could become unreadable outside a specific renderer. | DS files use ordinary prose and code-style notation, while the article generator emits explicit HTML for formulas, tables, and bibliography entries. | Approved |
| 2026-04-14 | DEC005 | Separate the generic engine from domain bundles. | The library should remain reusable beyond the initial package, sample, and manuscript experiments. | Domain-specific cues, invariants, and questions live in dedicated bundles; frontier and neighborhood logic remain generic. | Approved |
| 2026-04-14 | DEC006 | Make machine-readable experiment outputs authoritative. | The repository previously stored only hand-written experiment summaries, which made the claims impossible to reproduce. | Each experiment must write JSON and CSV outputs first, then derive `results.md` and SVG figures from them. | Approved |
| 2026-04-14 | DEC007 | Generate the article from current specifications and experiment outputs. | The article is only trustworthy if it can be rebuilt from the same artifacts that define and validate the implementation. | `docs/article/index.html` becomes a generated file driven by the reusable article-build skill over an article root with `index.html`, `assets/`, and `plan/`. | Approved |
| 2026-04-14 | DEC008 | Center the public API on complete analysis results. | Users need the full trace of hypotheses, theories, neighborhoods, and question updates, not only a winning label. | The top-level API should return structured analysis artifacts that can feed experiments and reporting directly. | Approved |
| 2026-04-14 | DEC009 | Allow optional AchillesAgentLib-backed LLM assistance. | The user explicitly authorized AchillesAgentLib and requested that all library-managed LLM interactions go through `LLMAgent`. | Optional ingestion normalization and conceptual explanation now share one dependency and one configuration model, while article review remains agent-owned rather than a repository LLMAgent path. | Approved |
| 2026-04-14 | DEC010 | Rebuild the article from chapter-plan files and generated chapter markdown. | The previous monolithic HTML builder made it too hard to reason about chapter intent, paragraph sourcing, and incremental rebuild conditions. | `docs/article/plan/plan.md`, `docs/article/plan/plan_chN.md`, generated chapter markdown files, declarative asset specs, and a reusable article-build skill become the new article workflow. | Approved |
| 2026-04-14 | DEC011 | Expand the validation corpus and add cue-masking robustness evaluation. | The earlier six-case corpus was too small to support the paper's stronger claims and did not test whether frontier retention remains useful when decisive cues are delayed or masked. | `data/inputs/workflow-cases.json` now carries an 18-case corpus, and `experiments/experiment3/` becomes the baseline-comparison and recoverability study used by the article. | Approved |
| 2026-04-14 | DEC012 | Make canonical CNL the stable public usage surface. | The new usage proposal requires a stable, auditable external contract that keeps source context, observation, theory, neighborhood, and update distinct instead of exposing only internal JavaScript objects. | The library now adds a usage-oriented evidence API and deterministic CNL serialization, and experiments must exercise that public usage surface rather than only internal helpers. | Approved |
