# DS003 - Experimental Protocol

## Introduction
The experiments are the validation layer of the project. They must demonstrate something more informative than final-label prediction. In the current repository, the experiments test three structural claims of the broader framework. First, richer observation should change the retained frontier before terminal evidence arrives. Second, explicit discriminating questions should reduce frontier uncertainty when several local theories remain plausible. Third, delayed commitment should remain useful when domain-specific cues are masked and early collapse policies become fragile. These studies do not validate the whole research program, but they must validate these structural claims cleanly and reproducibly.

## Core Content
The shared corpus currently contains eighteen controlled workflow cases across package handling, laboratory sample handling, and editorial manuscript processing. Every case is written as four ordered segments so the same case can be exposed through progressively longer prefixes. The dataset is deliberately engineered to mix overt domain cues with generic administrative language. This lets the experiments separate three regimes that matter for the paper: coarse early prefixes, typed mid-prefix cues, and residual ambiguity that still benefits from questioning.

Experiment 1 studies observer families. Each case is exposed through all four prefix depths, and every prefix is analyzed twice: once with the `coarse` observer and once with the `rich` observer. The authoritative outputs must record the retained frontier, the predicted domain distribution, frontier entropy, equivalence compression, question availability, and top-domain accuracy. This experiment should be interpreted as a representation test rather than a general intelligence benchmark. The central question is whether richer observation contracts the frontier earlier once typed cues appear.

Experiment 2 studies frontier narrowing through active questioning. It keeps the ambiguous traces that still expose a non-zero information-gain question after the initial analysis. For each retained trace, the library chooses the highest-information-gain question, applies the gold answer from the case metadata, and recomputes the frontier. The outputs must record the selected question, information gain, entropy before and after questioning, accuracy before and after questioning, and the surviving frontier structure. This experiment is a theory-management test: the point is not that every answer must finish induction immediately, but that the update makes the active frontier smaller, sharper, and easier to inspect.

Experiment 3 studies robustness under cue masking and compares the architecture against simpler baselines. The experiment keeps the full eighteen-case corpus but restricts attention to Prefix 2 and Prefix 3, where masking is most informative. Every trace is evaluated in clean form and in a deterministic cue-masked form that rewrites the most specific domain markers into more generic administrative language. The comparison policies are a lexical-support baseline over the base observational hypothesis, a single-best-theory baseline over the induced base theories, the immediate frontier top domain, the end-to-end frontier result after one question when available, and the retained-frontier truth rate. This experiment should be interpreted as a recoverability test: it shows whether preserving a frontier is better than collapsing early when the text suppresses or delays the decisive cue.

The metrics remain simple and reproducible. Accuracy is the proportion of traces where the selected policy returns the declared ground-truth domain. Frontier entropy is the Shannon entropy of the normalized retained-domain distribution. Equivalence compression is the ratio between retained theories and observer-relative equivalence classes. Question gain is the entropy drop induced by the selected question. Retained-frontier truth rate asks whether the ground-truth domain still survives anywhere on the frontier, and rescued rate asks how often a wrong pre-question top answer becomes correct after one discriminating answer.

Every experiment folder must contain authoritative narrative and machine-readable artifacts: a description Markdown file, structured JSON for the full run, CSV tables, SVG figures generated from those outputs, and a `results.md` report derived from the same data. When the corpus changes, or when any experiment is regenerated, the article must also be rebuilt so its tables, figures, and prose stay synchronized with the authoritative outputs.

The experiments should also validate the intended usage contract of the library rather than bypass it. They should invoke the high-level evidence-analysis and evidence-update entry points, record that the canonical CNL surface was used, and retain at least one representative CNL trace whenever question-driven update is part of the experiment. This keeps the evaluation aligned with the actual public usage model instead of measuring only internal helper functions.

## Conclusion
The project should claim validation only where the experiment scripts can regenerate the numbers, tables, and figures that appear in the article. Experimental prose is necessary, but it must be downstream of the data and honest about the scope of what the current controlled studies support.

### Critical Implementation Directives
1. Encode each case with explicit input text, prefix depth, observer family or masking condition, and declared ground truth.
2. Write machine-readable outputs first, then derive Markdown and SVG reports from them.
3. Keep the experiment scripts focused on orchestration and measurement; theory objects, frontier logic, question selection, and canonical usage serialization belong in reusable library modules.
